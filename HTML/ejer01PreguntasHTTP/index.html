  RESPUESTAS:

  • ¿Sobre que protocolo binario está montado el protocolo HTTP?

    El Protocolo http se encuentra montado sobre el protocolo TCP, el cual establece la conexión después de un handshaking de 3 vías (solicitud, aceptación y confirmación de conexión)

  • ¿Cuáles son los clientes http y los servidores http más utilizados?

    Los clientes son el navegador, el Curl(un software que consiste en una biblioteca y un interprete de comandos orientados a la transferencias de archivos), librerías nativas de diversos lenguajes de desarrollo. Algunos de los servidores web mas utilizados son Apache, Nginx, Microsoft IIS y LiteSpeed son algunos de los servidores más utilizados

  • ¿Que verbos admite un comando en el requerimiento HTTP?

    Los verbos o métodos admitidos por el requerimiento http es son:
    POST, CONNECT (uso en Proxys), DELETE, PUT, GET (solo lectura), PUT (utilizado para crear recursos en un servidor), DELETE (utilizado para borrar los recursos en un servidor), POST (utilizado Normalmente para actualizar recursos en un servidor).

  • ¿Que contenido lleva el body de un requerimiento HTTP?

    El body De un requerimiento puede contener datos en cualquier formato, ya sean datos ingresados en campos de un formulario, o archivos (files) en un proceso de subida (upload)

  • ¿Que diferencia existe entre un URL y un URI?

    Una URL es el subconjunto de URI. El Uri es lo que identifica el recurso y lo diferencia mediante un nombre u ubicación. En Cambio, la URL identifica la dirección web o la ubicación de un recurso único.

  • ¿Como se almacena información relacionada con las respuestas http?

    La información de un navegador se almacena en cookies una pequeña porción e datos que un servidor envía a el navegador web del usuario en un header de respuesta. El navegador almacena estos datos y los envía de regreso junto con la nueva petición al mismo servidor desde a la misma dirección de origen.

  • ¿Que significa Virtual hosting?

    Significa que puede haber varios dominios sobre una misma IP. Al usar hosts virtuales se declara el nombre de host en un servidor con DNS público. Normalmente esto se hostea por un ISP que tiene su propia zona de nombres de dominio.

  • ¿Por qué el HTTP se considera STATE LESS?

    Un protocolo StateLess (sin estado) es un protocolo en donde requerimiento se trata como una comunicación independiente y no tiene relación con cualquier requerimiento anterior. De esta manera se compone de pares independientes de solicitud y respuesta. Esto conlleva a que no sea necesario que el servidor retenga información sobre el estado de la comunicación durante las múltiples peticiones.

  • ¿Como va a mejorar la velocidad de la WEB en poco tiempo?

    Mediante HTTP/2 un protocolo Binario que conserva la semántica de la versión htpp1.1. Es nueva versión compacta cabeceras, priorizando elementos a transferir y multiplexando sin esperar las respuestas a los requerimientos. Esto resuelve gran cantidad de los problemas de velocidad que tiene la comunicación a través de TCP cerrada. Esto quiere decir que se puede enviar múltiples requerimientos sin tener que esperar respuestas y lo mismo a la inversa.

  • ¿Como va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?

    Actualmente es esta trabajando sobre el protocolo http3 que contiene un protocolo de conexión que reemplazara al TCP. Este se llama QUIC, el cual está montado con características del protocolo UDP para su velocidad y mantiene las conexiones seguras del protocolo TCP. Esto permite conseguir datos fiables, seguros y a mayor velocidad. Debido a esta mejora de velocidad, en las conexiones débiles o inestables en donde puede haber errores de transmisiones no se notará y parecerá más estable. Esto mejorara la recepción de paquetes, que por falta de confirmación de los receptores se generaban gran cantidad, lo que produce retardos en la transmisión de datos.
